一道小小的微信面试题

Posted on 2017-04-13

JavaScript异步编程大致有四种方式：

  1. 回调函数
  2. 时间监听
  3. 发布/订阅（观察者模式）
  4. Promise对象

问题描述

实现一个LazyMan，可以按照以下方式调用:
LazyMan(“Hank”)输出:
Hi! This is Hank!

LazyMan(“Hank”).sleep(10).eat(“dinner”)输出
Hi! This is Hank!
//等待10秒..
Wake up after 10
Eat dinner~

LazyMan(“Hank”).eat(“dinner”).eat(“supper”)输出
Hi This is Hank!
Eat dinner~
Eat supper~

LazyMan(“Hank”).sleepFirst(5).eat(“supper”)输出
//等待5秒
Wake up after 5
Hi This is Hank!
Eat supper
以此类推。

基本做法

使用队列+next. 

function LazyMan(name) {
    var queue = [];
    var task = {
        wait: function (second) {
            return function () {
                setTimeout(function () {
                    console.log('Wake up after ' + second);
                    next();
                }, second * 1000);
            };
        },
        eat: function (part) {
            return function () {
                console.log('Eat ' + part + '~');
                next();
            };
        },
        hi: function () {
            console.log('Hi! This is ' + name + '!');
            next();
        }
    };

    queue.push(task.hi);

    function next() {
        var fn = queue.shift();
        fn && fn();
    }

    // flush later
    setTimeout(function () {
        next();
    }, 0);

    return {
        sleep: function (second) {
            queue.push(task.wait(second));
            return this;
        },
        sleepFirst: function (second) {
            queue.unshift(task.wait(second));
            return this;
        },
        eat: function (part) {
            queue.push(task.eat(part));
            return this;
        }
    };
}

LazyMan('Hanks').sleepFirst(1).eat('dinner').sleep(10);

一些注意点：

  1. 源程序最后一行为module.exports = LazyMan;时可以这么测试；
    若最后一行为exports.Lazy = LazyMan;则测试为: 

    var Lazy = require('./LazyMan');
Lazy.LazyMan('Hanks').sleep(10).eat('breadfast');

  2. next()的使用，若任务队列为空，则不作处理；反之会执行函数。

    关于&&和||

    1. expr1 && expr2
      如果expr1 能转换成false则返回expr1,否则返回expr2. 因此, 在Boolean环境中使用时, 两个操作结果都为true时返回true,否则返回false.
    2. expr1 || expr2
      如果expr1能转换成true则返回expr1,否则返回expr2. 因此,在boolean环境(在if的条件判断中)中使用时, 二者操作结果中只要有一个为true,返回true;二者操作结果都为false时返回false.

总之，能确定这个逻辑表达式true or false的那一项将作为返回值。

  1. 程序中的下面这句是异步执行的重点，即在最后一句执行完毕后，此时所有任务都已经进入任务队列，进入下一轮Event Loop，触发next(). 
    setTimeout(function () {
     next();
    }, 0);
  2. 最后return语句中的return this;是为了链式调用。

ES6做法–byPromise

function createWaitPromise(second) {
    return new Promise(function (resolve, reject) {
        setTimeout(function () {
            resolve('Wake up after ' + second);
        }, second * 1000);
    });
}

function LazyMan(name) {
    var p = new Promise(function (resolve, reject) {
        resolve('Hi! This is ' + name + '!');
    });

    return {
        sleep: function (second) {
            p = p.then(function (msg) {
                console.log(msg);
                return createWaitPromise(second);
            });
            return this;
        },
        sleepFirst: function (second) {
            var op = p;
            p = createWaitPromise(second).then(function (msg) {
                console.log(msg);
                return op;
            });
            return this;
        },
        eat: function (part) {
            var pn = new Promise(function (resolve) {
                resolve('Eat ' + part + '~');
            });
            p = p.then(function (msg) {
                console.log(msg);
                return pn;
            });
            return this;
        },
        print: function () {
            return p.then(function (msg) {
                console.log(msg);
            });
        }
    };
}

module.exports = LazyMan;

测试：

var LazyMan = require('./LazyMan');
LazyMan('Hank').sleepFirst(10).eat('breadfast').sleep(5).eat('lunch').sleep(10).eat('dinner');

运行原理是把每一个任务的信息由下一个任务打印出来，最重要的是传递消息。程序精妙之处在于sleepFirst的编写，第一次写的时候写成下面这个样子： 

sleepFirst: function(seconds){
    createWaitPromise(seconds).then(function(msg){
        console.log(msg);
    });
    console.log(p);
    return this;
},

则下面的任务不会等待sleepFirst完成再执行，重点是把每一次的任务都绑定到p上.
